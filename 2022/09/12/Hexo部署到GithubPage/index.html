Hexo部署到GithubPage

安装Hexo

npm install -g hexo-cli

初始化hexo

cd 到博客文件夹
执行

hexo init

文件结构：

.

├── _config.landscape.yml

├── _config.yml

├── db.json

├── node_modules

├── package.json

├── package-lock.json

├── public

├── scaffolds

├── source

├── themes

└── yarn.lock

创建github.io repo

配置SSH Key

创建ssh key

ssh-keygen -t ed25519 -C "your_email@example.com

将提供的电子邮件地址为标签创建SSH密钥

将SSH密钥添加到 ssh-agent

1.后台启动ssh代理

eval "$(ssh-agent -s)"

Agent pid 59566

2.将 SSH 私钥添加到 ssh-agent（请将命令中的 id_ed25519 替换为私钥文件的名称）

ssh-add ~/.ssh/id_ed25519

在github setting中添加公钥

测试是否配置成功

ssh -T git@github.com

成功显示

Hi username! You’ve successfully authenticated, but GitHub does not
provide shell access.

编辑 _config.yml

添加
deploy:
type: git
repo:github.com:gitrepo地址
branch: main

写文章

hexo new (不接参数默认layout:post) "标题"

默认放在 /source/_post 目录下

更换主题

cd themes/

git clone xxxx(你喜欢的主题仓库)

vim _config.yml
themes: 主题名

常用命令

hexo clearn

hexo generate

hexo s (部署到本地测试)

hexo deploy （推送到git仓库）

挂梯子deploy会失败，先关闭。
